# Spec-Driven Development (SDD): Malayalam Music Randomizer (with Genres)

## 1. Product Overview
The platform is a web-based random music player that streams official Malayalam music videos, categorized by genre. It mimics the metadata-first accuracy of Chosic by decoupling track discovery from playback. Track metadata is grouped into distinct genres (e.g., Mollywood, Indie, Pop, Hip-Hop), sourced from a music database (Last.fm by default, Spotify as an optional alternative), mapped to the YouTube Data API to fetch official video IDs, and played via a randomized, minimal web interface with genre-filtering capabilities.

## 2. System Architecture
The application is separated into a serverless backend worker and a static frontend client. 

| Component | Technology | Responsibility |
|---|---|---|
| **Frontend** | HTML, CSS, JavaScript | Injects YouTube iframe, renders genre toggle buttons, handles randomizer logic per genre. |
| **Backend Worker** | Python (GitHub Actions) | Runs daily, loops through predefined genres, aggregates new tracks, maps metadata to YouTube. |
| **Primary Data Source** | Last.fm API | Provides free, user-tagged track metadata mapped to genres (`malayalam pop`, `mollywood`). |
| **Optional Data Source** | Spotify API | Provides strict official track metadata via genre search queries. |
| **Video Asset Source** | YouTube Data API v3 | Retrieves specific video IDs based on metadata queries. |
| **Database** | Static JSON (`songs.json`) | Acts as the bridge; structured as a dictionary (key = genre, value = array of IDs). |

## 3. API Integrations & Endpoints

To support genres, the backend script will contain a configured list of target tags/genres (e.g., `"mollywood"`, `"malayalam indie"`, `"malayalam pop"`, `"malayalam hip hop"`). The script will loop through this list during execution.

### 3.1 Last.fm API (Primary Track Discovery)
*   **Authentication:** Free API Key.
*   **Endpoint:** `tag.getTopTracks`
*   **Query Loop:** Executes once per genre.
    *   `tag`: `{current_genre_tag}`
    *   `limit`: `25` (Reduced limit per genre to save YouTube quota).
*   **Data Extracted:** Track Name, Artist Name, Target Genre.

### 3.2 Spotify API (Optional Track Discovery)
*   **Authentication:** Client Credentials Flow (Requires active Spotify Premium account).
*   **Endpoint:** `search`
*   **Query Loop:** Executes once per genre.
    *   `q`: `genre:"{current_genre_tag}" new` (e.g., `genre:"malayalam indie"`)
    *   `type`: `track`
    *   `limit`: `15`
    *   `market`: `IN`

### 3.3 YouTube Data API v3 (Video Mapping)
*   **Endpoint:** `search.list`
*   **Query Structure:** `"{Artist Name} - {Track Name} (Official Audio)"`
*   **Quota Management:** Script must cap at a combined **100 queries total** across all genres per day to strictly obey the 10,000-point daily free tier limit. (e.g., 4 genres × 25 tracks = 100 queries).

---

## 4. Data Flow & Automation

1. **Trigger:** `cron: '0 0 * * *'` executes the Python worker at midnight UTC.
2. **Fetch Phase:** Script loops through the predefined genre list. It connects to Last.fm (or Spotify) and downloads the latest track names for *each* genre.
3. **Mapping Phase:** Script iterates through the fetched lists. It pings the YouTube API to resolve the exact video ID for each track.
4. **Validation:** Script checks for duplicate video IDs within the specific genre bucket.
5. **Deployment:** The JSON object is updated and committed to the `main` branch, instantly updating the live frontend.
